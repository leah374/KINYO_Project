# 企业实践课程安排第四周以及Mid-Term任务
Author: Kai Wang, Yunxi Li, Yahui Li

## 前期数据评估与挑战
在初步数据处理及与企业方的沟通中，我们识别出首轮数据存在以下显著问题，导致难以支撑后续分析：
- 数据一致性低：数据与视频素材难以匹配，仅67条视频存在有效关联数据。
- 元数据质量差：素材标题存在大量乱码或无意义字符，缺乏语义信息。
- 统计口径混乱：数据来源混杂，缺乏统一标准。

## 解决方案与数据更新
针对上述问题，经与企业方协调，我们获取了更详实、规范的新数据集，具体情况如下：
- 时间跨度：覆盖2025年9月至2026年3月。
- 数据规模：包含3784条新视频记录-共708 GB。

## 当前成果
1. 数据处理与特征工程
- 数据集构建：对新获取的视频及元数据进行了深度清洗与预处理，最终构建了包含1742条有效且不重复的影片数据集。
- 视频流处理：利用FFMPEG对视频进行标准化压缩与关键帧截取。
- 处理策略：压缩旨在降低后续AI分析的Token消耗；同时，截取视频前15秒（Hook片段），以响应企业对于“黄金前三秒/前十五秒留存率”的关注，进行针对性分析。
2. 爆款样本筛选
- 筛选维度：基于3秒播放率、5秒播放率、10秒播放率、整体展现次数、整体点击次数这5项关键绩效指标进行综合评估。
- 筛选结果：成功识别出193条“爆款视频”，占有效数据集的11.1%，确立了高价值分析样本。
3. AI赋能与量化评估
- 模型应用：接入字节跳动火山引擎LAS（视频内容理解服务）。
- 评分机制：依据企业提供的《内容评价标准库.json》，调用豆包2.0-Pro大模型对193条爆款视频进行多维度自动化打分。
4. 统计分析与洞察
- 相关性分析：深入探究了AI打分结果与各业务指标之间的相关性，验证评价模型的有效性。
- 对比分析：对“爆款”与“非爆款”视频的各项指标进行了描述性统计，并绘制可视化图表，直观展示两者在数据表现上的显著差异。

## 分析结果
1. AI评分与留存指标的相关性验证
- 总体评分有效性：豆包模型给出的总分与3秒、5秒、10秒完播率均呈现显著的正相关性。这一结果有力证明了AI评分在衡量视频内容吸引力方面的可靠性与实用性。
- 细分维度复杂性：各细分维度的评分与完播率指标之间的相关性表现不一，方向不定。这表明需要进一步深入分析各维度的具体权重与计算方式，以优化评分模型。
2. 内容评分与流量指标的差异分析
- 低相关性发现：AI内容评分与视频的整体点击率、整体展示率相关性较弱。
- 原因推断：我们认为，点击率和展示率更多地受到平台推荐算法、外部元数据（如标题、标签）以及投流策略的影响，而非单纯由视频内容质量决定。
- 后续行动：计划向企业申请获取视频的标题、标签及投流等级等数据，以便进行更全面的归因分析。
3. 评价标准库的局限性探讨
- 当前局限：现有的《内容评价标准库.json》可能过度侧重于视频的技术质量，例如画面清晰度、音频质量等模态层面的因素。
- 内容本质：决定一个视频是否吸引人的核心要素，更多在于其内容本身，如幽默感、情感共鸣、叙事张力等，《内容评价标准库.json》中对于视频内容的评价部分较为粗糙，需要进一步精细化。
- 改进方向：现有的评价标准需要进行调整，以更准确地捕捉视频的内容价值。
4. 后续计划与资源协调
- 聚焦“黄金留存率”：针对企业高度关注的“黄金前三秒/前十五秒”留存率，我们将与企业进一步沟通，共同构建一套专门针对视频开头Hook的评分标准文件。
- API资源评估：企业已表示愿意提供OpenAI的API支持。但根据我们的初步实验，GPT-5.5在处理视频分析任务上的表现可能不及豆包2.0-Pro。后续将就模型选择、Token消耗及技术支持进行更深入的沟通。
